MercuPy is a Python Wrap for Mercury, the well known advanced n-body integrator used widely in planetary sciences studies.

The Mercury integrator was originally written by J.E. Chambers (Chambers, 1999) as an extension of the original and very succesful Swift Integrator designed by H. F. Levinson and M.J. Duncan (Levinson and Duncan, 1994).  Mercury uses simple albeit powerful algorithms to integrate the equation of motion of systems composed by n gravitationally interacting bodies.  It uses symplectic algorithms along with other complex and more precise integration methods, to evolve conservative systems while minimizing the numerical variation of total energy and angular momentum.  Mercury is able to integrate precisely the trajectories of the bodies when close encounters happen, something that the traditional fast symplectic integrators are unable to perform with the required precision.  It could also track the collisions among interacting bodies and compute the change in the axial tilt and angular momentum of rotation of the bodies.

Mercury was originally written in Fortran 77 and has a rather simple and unfriendly user interface.  The program is initialaized using a set of simple plain text files written in a very specific and hard to remember format.  The preparation of the input files, specially for large number of interacting bodies and the manipulation and analysis of the output data could be challenging.  For the newbie and even for advanced users, Mercury could be a hard to use piece of software.

 

MercuPy improves the experience using Mercury by providing:

  • An advanced and easy to configure user interface.
  • A set of python scripts to manipulate and represent final results.
  • A powerful plotting script to convert final data into high quality plots.

Other carachteristics added to Mercury by the MercuPy wrap includes but are not limited to:

  • The initial position of bodies can be described relative to an arbitrary reference frame using cartesian or classical elements respect to that frame of reference.
  • The final elements and state of the bodies can be translated and rotated to an arbitrary reference frame.
  • Once an integration has been completed you can continue it from the last state.
  • You can invert the direction of time and compare the results of a given integration with a previous trajectory in order to compute the precision of a given set of integrator parameters.
